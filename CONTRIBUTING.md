## ğŸ“ Writing Guidelines

### Research Notes (`/notes/`)

- Use clear, professional language
- Include technical details and explanations
- Connect concepts to the AI Brain architecture
- Reference related work and papers
- See template: `docs/templates/research-note-template.md`

### Paper Summaries (`/papers/`)

- Include full citation information
- Summarize key contributions
- Explain relevance to the AI Brain project
- Propose experiment ideas inspired by the paper
- See template: `docs/templates/paper-summary-template.md`

### Architecture Designs (`/systems-design/`)

- Use Mermaid diagrams for visualizations
- Specify module interfaces and interactions
- Document design decisions and trade-offs
- Include integration points with other modules

### Experiments (`/experiments/`)

- Define clear hypotheses
- Specify methodology and evaluation metrics
- Document results and analysis
- See template: `docs/templates/experiment-plan-template.md`

### Research Logs (`/research-log/`)

- Document daily/weekly discoveries
- Track insights and connections
- Note open questions and next steps
- See template: `docs/templates/research-log-template.md`
